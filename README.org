* soqonicatalogue

Go 1.25 microservice providing the product catalogue REST API for the [[https://github.com/abdvswmdr/soqoni][Soqoni]] e-commerce platform. Backed by MySQL with a many-to-many product/tag schema. Ships as a ~15MB Alpine Linux container via multi-stage build.

** Features

- Pure Go HTTP server — no framework overhead, sub-15MB production image
- Tag-based product filtering with many-to-many SQL schema
- Multi-stage Docker build: ~golang:1.25~ builder → ~alpine:3.19~ runtime
- Vendored dependencies — reproducible builds without network access
- Readiness and liveness probes for zero-downtime Kubernetes rolling updates
- Kubernetes-native: ClusterIP service, Ingress routing, HPA-ready

** API

| Endpoint | Method | Description |
|---|---|---|
| ~/catalogue~ | GET | List products (~?tags=~, ~?page=~, ~?size=~ supported) |
| ~/catalogue/{id}~ | GET | Get product by ID |
| ~/catalogue/size~ | GET | Total product count |
| ~/catalogue/tags~ | GET | All available tags |
| ~/health~ | GET | Liveness check |

#+BEGIN_SRC bash
# List all products
curl http://localhost:80/catalogue

# Filter by tag
curl 'http://localhost:80/catalogue?tags=iot'

# Health check
curl http://localhost:80/health
#+END_SRC

** Quickstart

#+BEGIN_SRC bash
docker run -p 8080:80 \
  -e ZIPKIN_ENABLED=false \
  -e DSN="user:pass@tcp(localhost:3306)/socksdb" \
  abdvswmdr/soqonicatalogue:latest
#+END_SRC

A MySQL 5.7 instance with the ~socksdb~ schema is required. See [[file:docker/soqonicatalogue-db][docker/soqonicatalogue-db]] for the database image with seed data.

** Environment Variables

| Variable | Default | Description |
|---|---|---|
| ~DSN~ | — | MySQL DSN: ~user:pass@tcp(host:3306)/db~ |
| ~PORT~ | ~80~ | Port this service listens on |
| ~ZIPKIN_ENABLED~ | ~false~ | Enable Zipkin distributed tracing |
| ~ZIPKIN_HOST~ | ~zipkin~ | Zipkin server hostname |

#+BEGIN_SRC bash

  -f docker/soqonicatalogue/Dockerfile-release .

cd soqonicatalogue
go mod download

# Run against a local MySQL instance
DSN="catalogue_user:pass@tcp(localhost:3306)/socksdb" go run cmd/cataloguesvc/main.go
#+END_SRC

Requires Go 1.25+.

Three tables in the ~socksdb~ database:

- ~sock~ — product records (id, name, description, price, count, image_url)
- ~tag~ — product tags
- ~sock_tag~ — junction table for the many-to-many product/tag relationship

** CI/CD

Jenkins pipeline (~Jenkinsfile~) runs on every push to ~main~:

1. *Test* — ~go test ./...~
2. *Build* — ~go build~ + ~docker build~ with git SHA tag
3. *Push* — ~docker push~ to [[https://hub.docker.com/r/abdvswmdr/soqonicatalogue][Docker Hub]]
