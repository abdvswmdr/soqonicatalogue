Demonstrates modern backend best practices: clean architecture (go-kit), enterprise observability, secure containerization, and Kubernetes-native design.

** Engineering Highlights

*** Clean Architecture Pattern
- **Go-kit framework**: Domain → Endpoint → Transport layer separation
- Service middleware stack: logging, tracing, and error handling baked in
- Endpoint-level circuit breakers (30s timeout) with Sony Gobreaker
- Type-safe error handling with structured error types

*** Enterprise Observability
- **Prometheus metrics**: ~request_duration_seconds~, ~inflight_requests~, request/response body sizes by route
- **Structured logging**: Timestamp, caller location, method execution time tracking
- **Distributed tracing**: OpenTracing integration ready (Jaeger/Zipkin compatible)
- ~GET /metrics~ endpoint for Prometheus scraping

*** Security & Compliance
- Non-root user execution (UID 10001) with restricted shell (~sbin/nologin~)
- Multi-stage Docker build: ~golang:1.25~ builder → ~alpine:3.19~ runtime (~15MB image)
- Vendored dependencies via ~go mod vendor~ — reproducible builds, no network dependency
- TLS-ready with proper error responses

*** DevOps Excellence
- Kubernetes-native: health probes, graceful shutdown, signal handling
- Environment-driven configuration (DSN, port, image path)
- Database connection pooling via sqlx with error recovery
- Zero-downtime deployment support

** API Specification

Standard REST endpoints with query parameter support for filtering, pagination, and multi-currency pricing.

| Endpoint | Method | Purpose | Query Parameters |
|---|---|---|---|
| ~/catalogue~ | GET | List products with optional filtering | ~?tags=~, ~?page=~, ~?size=~, ~?currency=~ (USD/EUR/KES), ~?order=~ |
| ~/catalogue/{id}~ | GET | Fetch product by ID with pricing | ~?currency=~ (USD/EUR/KES) |
| ~/catalogue/size~ | GET | Total product count | ~?tags=~ |
| ~/tags~ | GET | Available product tags | — |
| ~/health~ | GET | Liveness & readiness probe | — |
| ~/metrics~ | GET | Prometheus metrics export | — |
| ~/catalogue/{id}/stock~ | PUT | Decrement product stock | — |

*** Example Requests

#+BEGIN_SRC bash
# List all products with pagination
curl 'http://localhost/catalogue?page=1&size=10'

# Filter by tags and currency
curl 'http://localhost/catalogue?tags=iot,premium&currency=EUR'

# Product details with KES pricing
curl 'http://localhost/catalogue/507f1f77-bcf8-4fad-a26f-df9fdf8dcf0c?currency=KES'

# Service health check
curl http://localhost/health

# Prometheus metrics
curl http://localhost/metrics
#+END_SRC

** System Architecture

Three-tier microservice architecture with clear separation of concerns:

- **Transport Layer** (HTTP): Request/response marshaling, route matching, middleware chaining
- **Endpoint Layer**: Endpoint composition, circuit breaker injection, distributed tracing hooks
- **Service Layer**: Business logic, database queries, currency conversion

Database schema: normalized many-to-many design (sock ↔ sock_tag ↔ tag) for flexible tagging.

** Observability & Monitoring

*** Prometheus Metrics
Exposed at ~GET /metrics~ for Prometheus scraping:

- ~request_duration_seconds~: Histogram of request latency by method, route, status code
- ~inflight_requests~: Gauge of concurrent request count by method and route
- ~request_body_size_bytes~: Request payload size distribution
- ~response_body_size_bytes~: Response payload size distribution

*** Structured Logging
Every method invocation logs:
- Method name, parameters (tags, IDs, currency)
- Result count and execution time (microsecond precision)
- Error information with type

Example:
#+BEGIN_SRC
method=List tags="iot,premium" pageNum=1 pageSize=20 currency=EUR result=15 took=45.2ms
method=Get id=507f1f77-bcf8-4fad-a26f-df9fdf8dcf0c currency=USD took=12.1ms
#+END_SRC

*** Distributed Tracing
OpenTracing integration with per-endpoint span creation (Jaeger-compatible):
- All 6 endpoints emit traces with operation names: ~GET /catalogue~, ~GET /catalogue/{id}~, etc.
- TODO: Enable Zipkin collector with openzipkin-contrib update

** Security & Compliance

*** Container Security
- **Non-root execution**: UID 10001 (unprivileged user) with nologin shell
- **Minimal base image**: alpine:3.19 (~15MB) eliminates attack surface
- **No privilege escalation**: Strict USER directive in Dockerfile-release
- **Read-only artifacts**: Binary and image assets owned by service user

*** Reproducible Builds
- Vendored dependencies (~vendor/~) with locked transitive versions
- Go 1.25 with explicit module checksums (go.sum)
- Multi-stage build avoids shipping build tools to production
- Deterministic binary: ~CGO_ENABLED=0 GOOS=linux~

*** Error Handling
- Structured error types: ~ErrNotFound~, ~ErrDBConnection~
- Proper HTTP status codes: 404 for missing products, 500 for DB errors
- Sanitized error responses (no internal stack traces)

** Performance & Reliability

*** Circuit Breaker Pattern
Sony Gobreaker on every endpoint with 30-second timeout:
- Prevents cascading failures on database issues
- Auto-resets after cooldown period
- Applies to: List, Get, Count, Tags, DecrementStock endpoints

*** Connection Pooling
- sqlx.DB provides efficient MySQL connection reuse
- DB ping on startup validates connectivity (logs but doesn't fail)
- Configurable DSN supports connection string tuning

*** API Query Optimization
- Joins with GROUP_CONCAT for many-to-many product/tag relationships
- Indexed lookup by sock_id and tag_id
- Pagination support (offset/limit) for large result sets

** Development

*** Prerequisites
- Go 1.25+
- MySQL 5.7+
- Docker & Docker Compose (for local testing)

*** Setup
#+BEGIN_SRC bash
git clone https://github.com/abdvswmdr/soqonicatalogue
cd soqonicatalogue
go mod download

# Start MySQL with seed data
docker compose up -d catalogue-db

# Run service locally
DSN="catalogue_user:default_password@tcp(localhost:3306)/socksdb" go run cmd/cataloguesvc/main.go -port=8081

# Test endpoints
curl http://localhost:8081/catalogue
curl http://localhost:8081/health
#+END_SRC

*** Running Tests
#+BEGIN_SRC bash
# Unit tests
go test ./...

# Integration tests (requires docker)
docker compose -f docker-compose.yml up
go test -v -tags=integration ./...
#+END_SRC

*** Project Layout
- ~cmd/cataloguesvc/main.go~: Service initialization, flag parsing, middleware composition
- ~service.go~: Business logic, database queries, currency conversion
- ~endpoints.go~: Endpoint definitions with OpenTracing hooks
- ~transport.go~: HTTP handler, request/response marshaling, circuit breakers
- ~logging.go~: Middleware for structured logging
- ~docker/~: Multi-stage Dockerfile and database setup
- ~test/~: Integration test suites (Python + Docker)

** CI/CD Pipeline

Jenkins pipeline (~Jenkinsfile~) on every push to main branch:

1. **Build** — Compile dev image with Go toolchain
   - ~docker build -f docker/soqonicatalogue/Dockerfile .~

2. **Test** — Unit tests in containerized environment
   - ~docker run ... go test ./...~
   - ~docker compose~ integration tests

3. **Package** — Optimize release image
   - ~docker build -f docker/soqonicatalogue/Dockerfile-release .~
   - Apply git SHA as image tag

4. **Push** — Publish to Docker Hub
   - ~docker push abdvswmdr/soqonicatalogue:${GIT_COMMIT}~

Environment variables:
- ~GROUP~: Docker registry namespace (abdvswmdr)
- ~REPO~: Repository name (soqonicatalogue)
- ~COMMIT~: Git short SHA for versioning
